What is PGP?

Pretty Good Privacy (PGP) and GNU Privacy Guard (GPG) are encryption programs that follow the PGP standard and provide:

  • authentication, using signing
  • privacy, using encryption

Signing

PGP and GPG (from now on we'll just say "PGP") can use the private half of a PGP key to create a digital signature for any block of data:

  • text
  • source code
  • binary data
  • ... anything ...

A PGP signature can be validated by any PGP software.

All you need is:

  • the public half of the original key
  • the original data

That proves that the data hasn't been modified since the signature was created. And since only the person with the private half of the key could have created the signature, you can trust it's exactly the data they wanted you to see.

Encryption

PGP can use the public half of a PGP key to encrypt any block of data (all the same things it can sign) in a way that only someone with the private half of the key can decrypt it.

If someone encrypts something with the public half of your key, and only you have the private half of your key, then only you can decrypt and see that data.
